02 · 설계 원칙

커리큘럼 3가지 기준

1

기본기는 "리마인더" 수준으로, 사고법에 집중한다

타겟이 실무자이므로 재무제표 읽는 법 같은 기초는 30분 내 압축 정리. 강의의 무게 중심은 비즈니스 드라이버 설계와 합리적 가정 수립 능력에 둔다. 엑셀 함수·단축키는 다루지 않는다.

2

실습이 이론을 이끈다 — 전체의 68%가 핸즈온

개념 설명 → 즉시 AI 실습 → 결과 검토 → 판단 훈련의 루프로 구성. 각 챕터는 수강생이 실제 결과물(인터랙티브 모델 또는 검증된 모델)을 손에 쥐고 끝나도록 설계한다.

3

사고법 → AI 실습 → 업종 적용의 순차적 연결

"왜 이 구조인가"(사고법) → "AI로 어떻게 만드는가"(도구) → "내 업종에 어떻게 적용하는가"(실전)의 3단 계단이 끊기지 않도록 파트 간 연결을 명시적으로 설계한다.

P.3

업종별 AI 실전 모델링

4개 업종 × 2시간 — 실제 자료로 완성하는 인터랙티브 모델
8H 00M
Ch 3.1 · 2H

이커머스 / 플랫폼

GMV 기반 매출 드라이버 구조. 마케팅비-트래픽-전환율 연동 시뮬레이션. 실제 이커머스 IR 자료로 실습.

핵심 드라이버: 트래픽 → 전환율 → GMV → Take rate
Ch 3.2 · 2H

SaaS / 구독 서비스

ARR Waterfall + 코호트 기반 Churn 분석. MRR × 12 구조와 확장 MRR 개념. SaaS 공시 기업 자료 활용.

핵심 드라이버: 신규 MRR + 확장 MRR − Churn MRR
Ch 3.3 · 2H

제조 / 스타트업

원가 구조 분해(변동원가/고정비/인건비). 손익분기점(BEP) 시뮬레이션. 생산량 변동에 따른 마진 시나리오.

핵심 드라이버: 생산량 × (단가 − 변동원가) − 고정비
Ch 3.4 · 2H

유통 / 오프라인 서비스

점포별 매출 집계 구조. 매장 수 확장 시나리오와 신규점 램프업 모델. 인건비·임차료 고정비 압박 분석.

핵심 드라이버: 점포 수 × 점포당 매출 × 영업일
🔧

각 업종 챕터는 동일한 구조로 진행: ① 공시자료 분석 → ② Claude와 드라이버 대화 → ③ 인터랙티브 모델 생성 → ④ 가정 합리성 검토 → ⑤ 시나리오 비교. 수강생은 챕터마다 완성된 모델 파일을 손에 쥔다.

05 · 사용 도구

강의에서 사용하는 AI 스택

📊

Claude in Excel

워크북 전체 구조 인식, 셀 참조 추적, 공식 생성. 3-Statement Model 초안을 30분 만에. Pro 플랜($20/월)으로 사용 가능. 강의의 메인 실습 도구.

⚙️

Claude Code

사업계획서 업로드 → 인터랙티브 HTML 모델 생성. 슬라이더 시뮬레이션, 시나리오 탭, 브릿지 차트. 엑셀 없이 소통 가능한 모델을 만드는 핵심 도구.

🔍

Claude (Chat)

가정 설정을 위한 업종 벤치마크 리서치, 보고서 초안 작성, 드라이버 로직 대화. "이 가정이 합리적인가"를 함께 검토하는 싱킹 파트너.

📋

DART / SEC 공시자료

가상 기업이 아닌 실제 기업 자료로 실습. 국내는 DART, 해외는 SEC EDGAR에서 사업보고서·IR 자료 활용. 실전 감각 그대로.
